# AMR 使用手册（受众：营销专员·人）

> 适用版本 **v0.11.0** ｜ 更新日 2026-06-28
> 受众：客户的**营销专员（人）** + 配合的**营销数字专员（Agent）**。
> 设计原则：本软件 UI 力求**自明、不需手册**。本手册存在的每一段解释，都是一处 UI
> 自明性的"待改信号"——见文末「自明性审计」。理想状态：本手册趋近空白。

## 一页上手（理想情况，看界面即会）
打开收件箱（浏览器访问运维给你的内网地址），界面三栏，从左到右就是工作动线：

**左＝人** → **中＝会话** → **右＝事**

1. **左栏「📞 该联系谁」**：系统主动浮出该联络的人（🔴逾期 / ⭐你关注）。点一个人。
2. **中栏「会话」**：看你和 TA 的跨渠道完整对话（微信/电话/飞书合在一起）。只读，看清来龙去脉。
3. **右栏「事」**：处理这件事——
   - **🩺 诊断**：让 AI 看这段对话给个判断 + 一句话口径（该怎么回）。
   - **✨ 话术**：AI 按口径出 3 版（稳妥/直接/有温度），**「用此版」**挑一版（自动填入中栏回复框并高亮）。
   - 改好 → 中栏点 **「发送 →」** → **原地**弹出 **「✅ 确认发给 X / ✕ 改改」**，再点确认才真正发出（全文就在上方=发前再看一眼）。
   - **📌 承诺 / ✓ 办结**：跟踪这件事的承诺与完结。
   - **＋记一件事**：手动记一件待办/事项。

> **铁律**：系统**永远不替你发**。任何消息都要你亲手点「确认发送」。AI 只给草稿。

## 没有 AI 时（断网/未配模型）也能用
读消息、搜索、手敲回复、确认发送——全部照常。AI 只是助手：诊断/话术会显示"未配置"，
你手动写就是。**人始终在回路，能独立沟通。**

## 人在回路·人是主角（你必须知道的核心）

AI Agent 时代,人不被替代——**人机协同里你是主角,数字员工是放大器**。它替你跑腿、做更多杂活、提你的效率,但**它放大的是你、不是替代你**:创造和决策这些有价值的事永远是你做。

**自动沟通(分批上线)分三挡,由你自己往上拨**——系统**绝不自作主张升挡**:

| 挡 | 它怎么做 | 你怎么管 |
|---|---|---|
| **观察**(默认) | 只摆出"本来会自动发啥"，**一条不发** | 旁观，看它判得准不准 |
| **监管** | 它在安全区+时段内发，**发前给你倒计时可一键否决** | 旁看、随时刹车 |
| **自治** | 上班时段它自己跑 | **每 1-2 小时回来看简报**复盘、可全局刹车 |

**你的责任(也是你的价值):**
- **决策权不可交出去、不可偷懒。** 给它多少自治是你拨的；灌什么安全话术、群发名单审不审，是你的活。**AI 越能干，越要你掌好舵。**
- 它只自动发**"看不看都无害"**的(寒暄/确认/你批准的话术)；凡有分量的(承诺/金额/情绪)**永远等你拍板**。
- 它的**频率模仿真人能照顾的量**(每天几十条要思考的消息、有限客户)，**不刷屏**——是真照顾客户，不是机器群发。

> 一句话:**把杂活交给数字员工、把舵握在自己手里。** 这是 AMR 的人在回路最高准则，也是 AI 时代人依然有价值的地方。

## v0.8 这一版你会看到的新东西（操作 delta）

> 都在原来的三栏里，按界面提示点即可；这里只说"变了什么"。

- **发送是就地确认**：在中栏点 **「发送 →」**，原地变成 **「✅ 确认发给 X / ✕ 改改」**，不再跳到
  别的栏。发前全文就在上方，再看一眼再确认。
- **找对端点不用你操心**：一个人有好几个微信/电话号时，系统自动选当下发得出去的那个。
  若你想固定用某个号，让运维帮你**钉住首选端点**即可。
- **🩺 诊断 → ✨ 话术 顺序**：事卡里先点 🩺 诊断（看这段对话的问题 + 一句话口径），再点
  ✨ 话术出 3 版。话术区会提示"先诊断更准"。
- **话术带你的口吻**：AI 拟稿会模仿你自己平时发消息的语气，挑一版微调即可。
- **⚙ 设置**（新页/抽屉）：三件事都在这里，全部要你点确认才生效——
  - **自我身份**：把你本人的多个微信/电话/飞书号一次性勾选认领。认领后，**你自己发的消息显示
    为"我"**，你和自己/分身的对话不会再混进「📞 该联系谁」。
  - **人管理**：对某人 ⭐关注 / 🔗连渠道 / 🔓解绑（并错了一键拆开）。
  - **归一**：点 **🔄 启动归一** 让系统重新扫一遍把同一个人的多号合到一起；归并候选会列出
    **为什么匹配的证据**（电话/wxid）给你判断。**♻️ 复位归一** 需二次确认（只清机器自动连的，
    不动你手工确认过的）。

## 自明性审计（面向开发，非客户阅读）
写本手册时发现的 UI"待改信号"→ 已按自明性原则回改 UI：

- ✅ **已修**：旧「暂存待发→去最左栏确认」跨栏割裂 → 改为**就地内联确认**（中栏「发送→确认发给X/改改」），符合就近操作直觉。outbox 退为台账。
- ✅ **已修**：「用此版」跨栏因果 → 点击后自动**聚焦+高亮**中栏回复框，因果可见。
- ✅ **已修**：🩺诊断 应先于 ✨话术 → 话术区常驻提示"先🩺诊断更准"。
- ✅ 三栏「人→会话→事」动线本身自明，无需解释。

**v0.8 新发现（写本版 delta 时浮出的"待改信号"）：**
- ⚠️ **待改**：「⚙ 设置 → 自我身份」需要客户理解"认领自己的号"这件事——理想是**首启自动建议+
  勾选**，把"为什么要做"做成界面里一句话引导（如"认领后你发的消息显示为我，名单只剩客户"），
  而不是靠本手册解释。建议给"自我身份"加一行就地说明。
- ⚠️ **待改**：归并候选已展示"为什么匹配的证据"，但**「复位归一只清自动连」**这层语义仍需手册
  解释 → 应在二次确认弹窗里把"只会撤销机器自动合并、不动你手工确认过的"写进确认文案。
- ✅ **已修**：发送跨栏跳转 → 就地内联确认（中栏「发送→确认发给X/改改」），见上节。
- ⚠️ **观察**：「钉住首选端点」当前需运维代操作 → 若客户高频要换号，应把"设为首选端点"做成
  会话端点旁的一键操作，免去找运维。

> 方法论：本节从"待改清单"变成"已修清单"再生出新"待改"，正是「写手册=UI 自明性审计」回路
> 的持续产物——本版又少了一处跨栏割裂，又记下三处自明性欠账交还 UI。

## v0.10.0 新增（操作 delta）

> 默认落地页变了——打开就是**今日简报**，不再是三栏收件箱。以下说明新的操作动线。

### 打开即看简报（默认）

进入系统，默认看到**今日简报**（数字员工每天提交的工作报告），包含五张报告卡：

| 卡片 | 看什么 |
|---|---|
| 📈 销售报告 | 业务进行/成交/等待数 + 金额 |
| 📢 营销报告 | 大群触达 + 浮出的销售线索 |
| 🤝 关系报告 | 逾期🔴/转凉🟡 + 今日该主动联络谁 |
| 🗂 业务进展 | 事 lifecycle 推进 + 今日到期承诺 |
| 🧭 meta | 整体健康、异常、与北极星对齐 |

> **零 AI 也能看**：报告底层是 SQL 数字清单，无需 LLM 也正常显示数字和清单。
> 后端尚未落地的维度（如营销线索）显示"待后端"占位，不显示假数据。

### 需你拍板——一手势放行或改改

简报底部有「**需你拍板（N 项）**」条带，每一项都是系统建议、等你决定的动作（例如"撩这 4 个人"、"拉小群"）：

- 点 **「✅ 放行」**：同意，系统按建议执行（发送仍需二次确认，不会悄悄发出）。
- 点 **「✏️ 改改」**：打开对应条目调整再决定。

> 铁律不变：**放行 ≠ 自动发送**。任何对外消息仍需你在发送时点「确认发给 X」。

### 从简报钻进去（三层）

简报是发射台，不是终点。三层结构按需切换：
1. **L0 简报**（默认·看总结）—— 当前页
2. **L1 人/事双轴**（从某张报告卡点进去）—— 看该维度的人列表或事看板
3. **L2 会话/事全量**（再点进某人或某件事）—— 完整时间线，深看、决断

### 切换皮肤（简报 ⇄ 旧三栏收件箱）

习惯旧版三栏？右下角 **皮肤切换器** 可随时切：
- **简报**（默认）：今日简报 + 拍板条带
- **收件箱**：原来的左人/中会话/右事三栏

切换立即生效，刷新后仍记住你的选择（保存在浏览器本地）。**两套皮肤共用同一份数据，不丢任何内容。**

### 手机端使用

手机竖屏会自动切换到单列卡片流 + 底部标签栏，四个标签快速跳转：

| 标签 | 内容 |
|---|---|
| 📋 简报 | 今日简报 + 拍板条带 |
| 👥 人 | 联系人/关系列表 |
| 🗂 事 | 业务事件看板 |
| ⚙ 我 | 个人设置 |

操作手势和确认逻辑与桌面端完全一致——手机上"放行"同样是确认而非自动发送。

**v0.10.0 自明性审计新发现（面向开发）：**
- ⚠️ **待改**：「待后端」占位卡样式应与真实数据卡视觉区分明显（灰底或斜纹），避免客户误以为"数字是零"。
- ⚠️ **待改**：手机底部 tab「⚙ 我」与桌面「⚙ 设置」的功能范围应在入口旁加一行说明，否则客户不知道"皮肤切换"藏在哪。

## v0.11.0 新增（操作 delta · 自动回复怎么用 + 名字刷新）

> 这一版你会多看到一个 **🤖 自动回复** 面板。**默认全部关闭**——一条都不会自动发，
> 要不要让某个会话自动回、开到哪一挡，**全是你自己一个会话一个会话亲手拨的**。

### 🤖 自动回复面板：会话挡位怎么拨

每个会话现在多了一个**自动回复挡位**，三挡，你来拨（系统**永不替你升挡**）：

| 挡位 | 它会做什么 | 你看到/要做的 |
|---|---|---|
| **关**（默认） | **什么都不自动发** | 全靠你手敲回复——和以前一样 |
| **观察** | 只**摆出**"它本来会自动回啥"，**一条不发** | 旁观，看它判得准不准，零风险 |
| **监管** | 在安全话术 + 上班时段内，**准备好一条要发的回复，先给你倒计时** | 倒计时内你可**一键否决**，不否决才发 |

- **它只自动回"看不看都无害"的安全话术**（如"收到""好的"）——这些必须在你认可的**话术库白名单**里、
  且 AI 判为低风险，**两道闸同时过**才会进入"待发"。
- **凡有分量的**（承诺 / 报价 / 情绪安抚）→ 它**不自动发，直接交回你手敲**。
- 只在**上班时段（默认 9–21 点）**自动发，不会半夜打扰客户；频率**模仿一个真人能照顾的量**，不刷屏。

### 倒计时怎么否决 / 放行

会话进入「监管」挡后，当它准备好一条安全回复，你会看到一个**倒计时**（"X 秒后将发给 TA"）：

- **想拦下来** → 点 **「✕ 否决」**，这条就不发了（留痕记下你否决了）。
- **没问题** → 不用管，倒计时到了它**走和你手发一样的发送通道**（outbox + 确认留痕），不会凭空消失。

> **铁律不变**：哪怕在「监管」挡，**它发的每一条仍走和你一样的发送台账、留全痕**，
> 且你**始终有倒计时这道一键否决**。**系统永远不会绕过你偷偷发。**

### 群里的自动回（小群 / 大群 行为不同）

群聊的自动回有分寸，按群规模分两种行为，**照顾客户、不刷屏**：

- **小群（人少）或你是群主** → **积极回**（不需要别人@你也会回，主动维护关系）。
- **大群** → **只有别人@到你**才回（防止在大群里刷屏扰人）。
- 拿不准的群 → **保守**（当大群处理，@你才回）。

> 说明：当前"群里有多少人"是按**群里发过言的人数**估的（偏少），等渠道后端给出权威群名册后会更准——
> 这不影响你用，只是边界群的分挡可能略保守。

### 名字刷新（联系人显示名对不上时）

如果某个联系人在系统里显示的名字过时了（对方改了群昵称/备注），可以让系统**从最新会话名回填**：
- 这是个**先预览、后确认**的操作（先给你看"要把 A 改成 B"，你确认了才真改）。
- 高频要改名时，让运维把这个入口做到联系人卡旁；目前可请运维代跑一次。

**v0.11.0 自明性审计新发现（面向开发）：**
- ⚠️ **今天踩到·待改**：**会话挡位下拉要先选中某个会话才会亮**——直接想拨挡时它是灰的，容易让人以为坏了。
  应在未选会话时给挡位控件一句就地提示（"先在中栏选一个会话再拨挡"），或选会话后高亮一下挡位控件。
- ⚠️ **待改**：「观察」挡的"它本来会发啥"应和真实已发消息在视觉上明显区分（如加"草稿/未发"水印），
  避免营销专员误以为这条已经发出去了。
- ⚠️ **待改**：倒计时否决窗应有**声音/醒目提示**，否则人离开屏幕时可能错过否决时机（虽不影响安全——
  发的都是白名单安全话术——但体验上应让人"想拦就拦得住"）。
