# AMR 使用手册（受众：营销专员·人）

> 适用版本 **v0.10.0** ｜ 更新日 2026-06-28
> 受众：客户的**营销专员（人）** + 配合的**营销数字专员（Agent）**。
> 设计原则：本软件 UI 力求**自明、不需手册**。本手册存在的每一段解释，都是一处 UI
> 自明性的"待改信号"——见文末「自明性审计」。理想状态：本手册趋近空白。

## 一页上手（理想情况，看界面即会）
打开收件箱（浏览器访问运维给你的内网地址），界面三栏，从左到右就是工作动线：

**左＝人** → **中＝会话** → **右＝事**

1. **左栏「📞 该联系谁」**：系统主动浮出该联络的人（🔴逾期 / ⭐你关注）。点一个人。
2. **中栏「会话」**：看你和 TA 的跨渠道完整对话（微信/电话/飞书合在一起）。只读，看清来龙去脉。
3. **右栏「事」**：处理这件事——
   - **🩺 诊断**：让 AI 看这段对话给个判断 + 一句话口径（该怎么回）。
   - **✨ 话术**：AI 按口径出 3 版（稳妥/直接/有温度），**「用此版」**挑一版（自动填入中栏回复框并高亮）。
   - 改好 → 中栏点 **「发送 →」** → **原地**弹出 **「✅ 确认发给 X / ✕ 改改」**，再点确认才真正发出（全文就在上方=发前再看一眼）。
   - **📌 承诺 / ✓ 办结**：跟踪这件事的承诺与完结。
   - **＋记一件事**：手动记一件待办/事项。

> **铁律**：系统**永远不替你发**。任何消息都要你亲手点「确认发送」。AI 只给草稿。

## 没有 AI 时（断网/未配模型）也能用
读消息、搜索、手敲回复、确认发送——全部照常。AI 只是助手：诊断/话术会显示"未配置"，
你手动写就是。**人始终在回路，能独立沟通。**

## v0.8 这一版你会看到的新东西（操作 delta）

> 都在原来的三栏里，按界面提示点即可；这里只说"变了什么"。

- **发送是就地确认**：在中栏点 **「发送 →」**，原地变成 **「✅ 确认发给 X / ✕ 改改」**，不再跳到
  别的栏。发前全文就在上方，再看一眼再确认。
- **找对端点不用你操心**：一个人有好几个微信/电话号时，系统自动选当下发得出去的那个。
  若你想固定用某个号，让运维帮你**钉住首选端点**即可。
- **🩺 诊断 → ✨ 话术 顺序**：事卡里先点 🩺 诊断（看这段对话的问题 + 一句话口径），再点
  ✨ 话术出 3 版。话术区会提示"先诊断更准"。
- **话术带你的口吻**：AI 拟稿会模仿你自己平时发消息的语气，挑一版微调即可。
- **⚙ 设置**（新页/抽屉）：三件事都在这里，全部要你点确认才生效——
  - **自我身份**：把你本人的多个微信/电话/飞书号一次性勾选认领。认领后，**你自己发的消息显示
    为"我"**，你和自己/分身的对话不会再混进「📞 该联系谁」。
  - **人管理**：对某人 ⭐关注 / 🔗连渠道 / 🔓解绑（并错了一键拆开）。
  - **归一**：点 **🔄 启动归一** 让系统重新扫一遍把同一个人的多号合到一起；归并候选会列出
    **为什么匹配的证据**（电话/wxid）给你判断。**♻️ 复位归一** 需二次确认（只清机器自动连的，
    不动你手工确认过的）。

- ✅ **已修**：旧「暂存待发→去最左栏确认」跨栏割裂 → 改为**就地内联确认**（中栏「发送→确认发给X/改改」），符合就近操作直觉。outbox 退为台账。
- ✅ **已修**：「用此版」跨栏因果 → 点击后自动**聚焦+高亮**中栏回复框，因果可见。
- ✅ **已修**：🩺诊断 应先于 ✨话术 → 话术区常驻提示"先🩺诊断更准"。
- ✅ 三栏「人→会话→事」动线本身自明，无需解释。

- ⚠️ **待改**：「⚙ 设置 → 自我身份」需要客户理解"认领自己的号"这件事——理想是**首启自动建议+

  解释 → 应在二次确认弹窗里把"只会撤销机器自动合并、不动你手工确认过的"写进确认文案。
- ✅ **已修**：发送跨栏跳转 → 就地内联确认（中栏「发送→确认发给X/改改」），见上节。

  会话端点旁的一键操作，免去找运维。

## v0.10.0 新增（操作 delta）

> 默认落地页变了——打开就是**今日简报**，不再是三栏收件箱。以下说明新的操作动线。

### 打开即看简报（默认）

进入系统，默认看到**今日简报**（数字员工每天提交的工作报告），包含五张报告卡：

| 卡片 | 看什么 |
|---|---|
| 📈 销售报告 | 业务进行/成交/等待数 + 金额 |
| 📢 营销报告 | 大群触达 + 浮出的销售线索 |
| 🤝 关系报告 | 逾期🔴/转凉🟡 + 今日该主动联络谁 |
| 🗂 业务进展 | 事 lifecycle 推进 + 今日到期承诺 |
| 🧭 meta | 整体健康、异常、与北极星对齐 |

> **零 AI 也能看**：报告底层是 SQL 数字清单，无需 LLM 也正常显示数字和清单。
> 后端尚未落地的维度（如营销线索）显示"待后端"占位，不显示假数据。

### 需你拍板——一手势放行或改改

简报底部有「**需你拍板（N 项）**」条带，每一项都是系统建议、等你决定的动作（例如"撩这 4 个人"、"拉小群"）：

- 点 **「✅ 放行」**：同意，系统按建议执行（发送仍需二次确认，不会悄悄发出）。
- 点 **「✏️ 改改」**：打开对应条目调整再决定。

> 铁律不变：**放行 ≠ 自动发送**。任何对外消息仍需你在发送时点「确认发给 X」。

### 从简报钻进去（三层）

简报是发射台，不是终点。三层结构按需切换：
1. **L0 简报**（默认·看总结）—— 当前页
2. **L1 人/事双轴**（从某张报告卡点进去）—— 看该维度的人列表或事看板
3. **L2 会话/事全量**（再点进某人或某件事）—— 完整时间线，深看、决断

### 切换皮肤（简报 ⇄ 旧三栏收件箱）

习惯旧版三栏？右下角 **皮肤切换器** 可随时切：
- **简报**（默认）：今日简报 + 拍板条带
- **收件箱**：原来的左人/中会话/右事三栏

切换立即生效，刷新后仍记住你的选择（保存在浏览器本地）。**两套皮肤共用同一份数据，不丢任何内容。**

### 手机端使用

手机竖屏会自动切换到单列卡片流 + 底部标签栏，四个标签快速跳转：

| 标签 | 内容 |
|---|---|
| 📋 简报 | 今日简报 + 拍板条带 |
| 👥 人 | 联系人/关系列表 |
| 🗂 事 | 业务事件看板 |
| ⚙ 我 | 个人设置 |

操作手势和确认逻辑与桌面端完全一致——手机上"放行"同样是确认而非自动发送。

**v0.10.0 自明性审计新发现（面向开发）：**
- ⚠️ **待改**：「待后端」占位卡样式应与真实数据卡视觉区分明显（灰底或斜纹），避免客户误以为"数字是零"。
- ⚠️ **待改**：手机底部 tab「⚙ 我」与桌面「⚙ 设置」的功能范围应在入口旁加一行说明，否则客户不知道"皮肤切换"藏在哪。
