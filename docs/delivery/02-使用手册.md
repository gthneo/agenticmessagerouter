# AMR 使用手册（受众：营销专员·人）

> 适用版本 **v0.7** ｜ 更新日 2026-06-16
> 受众：客户的**营销专员（人）** + 配合的**营销数字专员（Agent）**。
> 设计原则：本软件 UI 力求**自明、不需手册**。本手册存在的每一段解释，都是一处 UI
> 自明性的"待改信号"——见文末「自明性审计」。理想状态：本手册趋近空白。

## 一页上手（理想情况，看界面即会）
打开收件箱（浏览器访问运维给你的内网地址），界面三栏，从左到右就是工作动线：

**左＝人** → **中＝会话** → **右＝事**

1. **左栏「📞 该联系谁」**：系统主动浮出该联络的人（🔴逾期 / ⭐你关注）。点一个人。
2. **中栏「会话」**：看你和 TA 的跨渠道完整对话（微信/电话/飞书合在一起）。只读，看清来龙去脉。
3. **右栏「事」**：处理这件事——
   - **🩺 诊断**：让 AI 看这段对话给个判断 + 一句话口径（该怎么回）。
   - **✨ 话术**：AI 按口径出 3 版（稳妥/直接/有温度），**「用此版」**挑一版。
   - 改好 → **「暂存待发」** → 去左栏「📤 待发送」**「✅ 确认发送」**才真正发出。
   - **📌 承诺 / ✓ 办结**：跟踪这件事的承诺与完结。
   - **＋记一件事**：手动记一件待办/事项。

> **铁律**：系统**永远不替你发**。任何消息都要你亲手点「确认发送」。AI 只给草稿。

## 没有 AI 时（断网/未配模型）也能用
读消息、搜索、手敲回复、确认发送——全部照常。AI 只是助手：诊断/话术会显示"未配置"，
你手动写就是。**人始终在回路，能独立沟通。**

## 自明性审计（面向开发，非客户阅读）
写本手册时发现的 UI"待改信号"（每条＝需口头解释＝UI 不够自明，应回改 UI）：

- ⚠️ **「暂存待发」→ 去左栏确认** 是跨栏两步，需要解释"为什么不在原地发"。
  （王总已定：保留两步=跨会话复核+误点缓冲；但措辞/视觉可再强化引导，降低解释成本。）
- ⚠️ **「用此版」（右栏）填入的是中栏回复框**，跨栏因果需解释。候选：点击时高亮目标框/加一帧动效。
- ⚠️ **🩺诊断 与 ✨话术 的先后**（先诊断出口径再拟话术更准）当前靠用户自悟。候选：话术按钮在无诊断时提示"先诊断更准"。
- ✅ 三栏「人→会话→事」动线本身自明，无需解释。
