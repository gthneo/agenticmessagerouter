# AMR 运维手册（受众：数字运维工程师 Agent + 现场 Build 工程师）

> 适用版本 **v0.8** ｜ 更新日 2026-06-16
> 写法：面向 **Agent 可直接执行**——命令块 + 排障表 + 凭据路径。**凭据不入库**：
> SSH/token 等真值见运维方本地笔记（不在公开仓库）。部署主机 IP/端口为内网默认。

## 部署 / 升级
```sh
# 从开发机推代码 + 刷 systemd --user 服务（幂等）
deploy/deploy.sh <user>@<deploy-host>
# 等价手动：rsync src/jl pyproject.toml deploy → ~/amr/，再 bash ~/amr/deploy/install.sh
```
- 零运行时依赖（stdlib），系统 python3.10 跑，`PYTHONPATH=src`。
- DB：`~/.config/jl/jl.db`（真相源，**不入库**）。Schema 变更经 `db.init_db` 幂等 ALTER
  (`_ADDED_COLUMNS`) + 新表 `CREATE TABLE IF NOT EXISTS` 自动迁移，无需手动 migrate。

## 服务（systemd --user，enable-linger 重启存活）
```sh
systemctl --user status|restart amr-web amr-poll
journalctl --user -u amr-web -n 100 --no-pager
```
- `amr-web`：只读+操作 web 收件箱，:8088（token 在 `~/.config/jl/web_token`）。
- `amr-poll`：5 分钟拉新 + 自动拟稿 sweep + 主动 opener sweep（均 LLM-optional，永不自动发）。
- 环境：`~/.config/jl/amr.env`（`JL_WEB_TOKEN` / `AMR_LLM_PROVIDER=claude_code` /
  `AMR_CLAUDE_BIN`=绝对路径）。install.sh 首次生成 web_token + 写 amr.env。

## 常用运维命令（jl CLI）
```sh
jl                      # 关系账户健康度 sweep（染色红榜）
jl 主动 [名]            # 主动联络队列 / 给单人拟开场
jl 关注 [--off] <名>    # 标记/取消关注（进主动队列）
jl 连 <名> <chat_id>    # 把人连到正确的实时微信会话（修缺渠道/换正确端点）
jl 关注 [--off] <名>    # 标记/取消关注（进主动队列）
jl 电话归一             # 折叠同号 +86/格式变体重复会话；号码 canon 去重
jl 解绑 <conv>          # 拆开错误归并（把某会话从当前 person 解开，归一 v2）
jl --tokens            # LLM token 用量统计
```
端点维护（python，按需）：`db.dedup_channels(c)`（折叠渠道表同号端点）、
`db.dedup_phone_conversations(c)`（折叠会话表）、`db.set_endpoint_pin(c, pid, kind, ident, True)`（钦点首选发送端点）。

## 自我身份 + 归一（v0.8，全人在回路）
```sh
# 自我身份：首启自动从 accounts.self_id 种子；建议+勾选见 ⚙ 设置 / API
#   db.seed_self_from_accounts(c)        # 把 accounts.self_id 纳入登记（persona=自我）
#   db.suggest_self_identities(c)        # 自动建议候选（不自动入，人勾选）
#   db.add_self_identity(c, kind, ident, persona='自我', label='')
#   db.apply_self_directions(c)          # 把 sender∈self 的消息标 direction=out（"我"），ingest 后调，幂等
# 归一（reunify）：
#   db.reunify(c)                        # 全扫：强信号自动连 + 弱信号留候选
#   db.reunify(c, reset=True)            # 复位：只清【自动连】（不动人工确认），重跑——需二次确认
```
- 自我会话（`chat_id ∈ self_identities`）被 `proactive_sweep`/openers/`persons_overview`
  **自动跳过**——不给自己拟开场、不进「该联系谁」/联系人。
- Web 设置 API（人在回路，留痕 events）：`GET/POST /api/self`、`POST /api/self/remove`、
  `POST /api/connect`、`POST /api/unlink`、`POST /api/reunify`（`?reset=1` 需二确）。
- **registry seeded from account self_id**：渠道命令的本人身份从 `accounts.self_id` 自动播种，
  不手填真值（凭据/真值不入库）。

## 排障表（dogfood 实战）
| 症状 | 原因 | 处理 |
|---|---|---|
| 发送「No action selected」/「TA 不在微信近期会话」 | 目标 chat_id 不在 fullwechat 实时可选列表（裸 wxid/已冷） | 在微信里打开与 TA 的对话使其进近期列表，或 `jl 连 <名> <实时id>` 重连正确端点 |
| 时间线出现 `<msg>cdnthumb…` 原始 XML | 旧数据媒体未清洗 | 用 `fullwechat.clean_content` 按 raw 重算 content（见 git 历史的清洗脚本）；新数据 ingest 已自动清 |
| 一人多条重复电话/渠道 | +86/格式变体未归一 | `jl 电话归一` + `db.dedup_channels` |
| 两个人被错并成一个 | 归一 v2 误连/弱信号被接受 | `jl 解绑 <conv>` 拆开；或 `db.reunify(c, reset=True)`（二确，只清自动连） |
| 本人出现在「该联系谁」/给自己拟了开场 | 自我身份未声明 | ⚙ 设置勾选认领，或 `db.add_self_identity`；`db.apply_self_directions(c)` 重标方向 |
| AI 话术/诊断「未配置 LLM」 | claude CLI 未登录 / amr.env 缺 AMR_CLAUDE_BIN | 检查 `~/.config/jl/amr.env`；`claude` 须以运维账号登录（Max plan）；人手操作不受影响 |
| 发送被拒（auto 模式拦截） | HITL 闸：禁止 Agent 代发 | 正常——由人在 web UI 点「确认发送」 |
| git push 卡住 | 受限网络 HTTPS 上传慢，勿用短超时掐断 | `GIT_HTTP_LOW_SPEED_TIME=20 git push`，给 60s+，别自掐 |

## 凭据轮换
- web token：删 `~/.config/jl/web_token` 后重跑 install.sh 重生成。
- fullwechat / claude / lark 凭据：见运维方本地笔记，按各渠道流程轮换；**不写入仓库**。

## 监控
- `journalctl --user -u amr-poll` 看每轮 `+N` 拉新与异常；`jl --tokens` 看 LLM 消耗。
- reboot 存活：`loginctl enable-linger <user>` 已开（install.sh 设）。
