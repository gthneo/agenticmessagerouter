# AMR 开发日志 · 2026-06-17 ｜ v0.6 → v0.9.1

## 总
AMR 从"会话级回复助手"进化成一个**真正的 Message Router**：会话 → **端点**(一人多渠道每渠道多号) → **多工具**(同一渠道多个接入后端) → **跨账号归一**(同一个人在你多个自有账号上合一)。真机 dogfood 已把**一个真实联系人在 3 个账号 × 2 个工具上收成一条线**。

## 里程碑（分）
- **v0.7 关系智能·人和事谐**：Endpoint 路由模型（按 渠道权重×新鲜度×可发性 选最优端点 + 人钦点 + 降级）；**人 / 会话 / 事 三栏 UI**（事卡 = 诊断→话术→人审发→承诺）；电话号归一（去 +86 重复）。
- **v0.7 沟通教练·外圆内方**：**T4 诊断引擎固化**（本地 rubric + LLM 结构化打分 + 人审）；核心博弈原则 **以牙还牙、以蜜还蜜**（tit-for-tat）；诊断口径驱动起草。**口吻沉淀**：话术模仿你本人已发消息的语气。
- **v0.8 SELF 自我归一**：用户本人=中心节点（不是联系人），多渠道多身份、persona 面具、出站识别（你发的标「我」）、自我会话排除出主动/开场。
- **v0.9 可扩展多工具消费层**：`accounts.tool` + 能力契约（读/发）；**PowerDataAdapter**（MCP-over-HTTP 接入只读微信工具）。
- **v0.9.1 跨账号归一**：PowerData `get_contacts` 取 wxid → 按 wxid 把同一个人在多账号/多工具合成一个 person；官方号过滤；同人多 wxid 人工确认合并。
- **归一硬化**：候选按信号分级 + 证据可见（电话/wxid/为什么匹配）；解绑误并；「这其实是我」一键设自我。

## 真机验证（dogfood）
- **某真实联系人 三账号合一**：微信×2(两个 wxid) + 电话×2，横跨 fullwechat / PowerData / CallHistory 三工具、三账号 → **一个 person**。
- **一条高风险危机会话**只存在于"某工具独占"的自有账号上，另一工具完全看不到 → 接入 PowerData 后 AMR 才吃得进来。这是"同一渠道多工具"必须存在的铁证。

## 钦定的架构原则（沉淀）
1. **同一渠道可有多种接入工具** —— 这是 Message Router 的定义性特征。
2. **Agentic AI Era：消费数据的接口必须可扩展** —— ingest 层是可插拔开放抽象（任何新渠道/工具实现 adapter 契约即接入）。
3. **使用手册 = UI 自明性的校验器**（要解释的地方就是 UI 待改）；交付/运维文档给 Agent 看和用。
4. **人在回路 + LLM-optional + 公开仓库洁净** 贯穿始终。

## 工程纪律
- **268 项测试全绿**；纯逻辑 TDD，渠道/网络靠真机验证。
- **多 Agent 并行加速**：按文件分工（如 web ‖ docs、消费层 ‖ 适配器），零冲突合并。

- **HITL 闸多次拦下"代发"**（敏感消息必须人确认才发）。
- 4 套客户端交付文档随版更新（营销/使用/培训/运维，受众分流）。
