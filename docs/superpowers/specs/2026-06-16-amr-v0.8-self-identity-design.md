# AMR v0.8 — SELF(自我)归一 + 归一/端点/自我上界面 — Design

**Date:** 2026-06-16 ｜ **Status:** approved (brainstorm, 质疑两轮收敛) ｜ 受 [[self-identity-and-personas]] 记忆约束。

## Goal
把"用户本人"立为 AMR 的**中心节点（不是联系人）**：SELF 多渠道×每渠道多身份、扁平、带
persona 面具标签。自我归一 → 出站识别 + 自我会话归「我的」区、排除出主动/开场。并把
归一/端点/自我 的人在回路操作**搬上 web 设置界面**。养心博弈区留下批。

## Decisions (王总 2026-06-16, 质疑两轮)
- SELF = **独立自我身份登记**（不是 person）；多身份**扁平不分层**。
- 每身份打 **persona 面具**；默认极简 **{自我 / AI分身 / 经营}**，可增改。
- 自我会话 = persona 间博弈（养心地基，下批）；**渠道不变、不跨渠道沟通/合并**。
- 声明 = **自动建议 + 勾选**（人在回路；机器不擅判 self）。
- 启用时**自动全扫+归一**（可设置开关）；**复位归一二次确认**。
- 归一/端点/自我 操作上 **web 设置界面**（更新式界面）。

## Architecture

### 1. self_identities 登记表（schema.sql 唯一 DDL 源）
```sql
CREATE TABLE IF NOT EXISTS self_identities (
    id         INTEGER PRIMARY KEY,
    kind       TEXT NOT NULL,                 -- wechat/phone/feishu/xhs/wecom...
    identifier TEXT NOT NULL,                 -- wxid / 电话(canon) / open_id ...
    persona    TEXT NOT NULL DEFAULT '自我',   -- 自我 / AI分身 / 经营 (可改)
    label      TEXT NOT NULL DEFAULT '',
    created_at INTEGER NOT NULL,
    UNIQUE (kind, identifier)
);
```
新表，旧库经 `CREATE TABLE IF NOT EXISTS` 自动建（init_db）。

### 2. db ops
- `add_self_identity(conn, kind, identifier, persona='自我', label='')`（phone 存 canon）。
- `get_self_identities(conn)` / `remove_self_identity(conn, kind, identifier)`。
- `is_self(conn, kind, identifier)` → bool（phone 比 canon）。
- `seed_self_from_accounts(conn)`：把 `accounts.self_id` 自动纳入登记（persona=自我）。
- `suggest_self_identities(conn)`：自动建议——accounts.self_id + 名字含「我/本人」或等于用户真名
  的会话 peer → 候选给人勾选（不自动入）。

### 3. 出站识别（解决 direction 写死=in）
`apply_self_directions(conn)`：把 `messages.sender_id ∈ self(wechat identifiers)` 的消息
`direction='out'`（"我"）。ingest 后调用（LLM 无关、幂等）。phone 的 "me" 已由 map_call 标。

### 4. 自我会话归类 + 排除
- 一条会话是「自我」当其 `chat_id ∈ self_identities`（同渠道 identifier 命中）。
- `proactive_sweep` / `api_proactive` / `persons_overview` / openers **跳过自我会话**
  （绝不给自己拟开场、不进 该联系谁/联系人）。
- 自我会话进「🪞 我的」区（v0.8 仅列出+标 persona；persona 间博弈/养心区下批）。

### 5. web 设置界面（归一/端点/自我 上界面）
新页 `/settings`（或主页加「⚙ 设置」抽屉），含三块，全人在回路：
- **自我身份**：`GET /api/self`（列已登记 + 建议）、`POST /api/self`（勾选加，带 persona）、
  `POST /api/self/remove`。展示 渠道/标识/persona。
- **人管理**：对某人 ⭐关注 / 🔗连渠道(`POST /api/connect`) / 🔓解绑(`POST /api/unlink`)。
  （把现有 CLI 能力搬上界面。）
- **归一控制**：「🔄 启动归一(全扫+link+suggest)」`POST /api/reunify`；
  「♻️ 复位归一(解所有自动连重跑)」`POST /api/reunify?reset=1` **需二次确认**；开关：启用时自动归一。

### 6. 启动/复位归一
- `reunify(conn, *, reset=False)`：reset 时先把**自动连**(非人工确认)的 person_id 清空，再
  `link_conversations` + 留 `suggest_merges` 候选。返回统计。人工确认的链接不动（reset 只清自动）。
- 启用自动归一：install/首启时跑一次 `seed_self_from_accounts` + `reunify`（开关存配置）。

## 数据流
ingest → `apply_self_directions`(标我) → `reunify`(强信号自动连/弱信号候选) → 自我会话归「我的」、
排除主动 → 联系人/该联系谁/事卡 只剩真·他人。设置界面人工：声明自我(建议+勾选)、管人、触发/复位归一。

## 安全 / 原则
- 人在回路：self 声明、连渠道、解绑、复位归一 全人工确认/二确，留痕 events。
- 不跨渠道沟通 self（渠道不变，渠道内对应）。
- LLM 无关：本批纯确定性 + UI，无模型依赖。
- 复位只清**自动连**，不动人工确认的归一（不毁人工劳动）。

## 测试（合成 fixtures）
- db：add/get/remove/is_self（phone canon）、seed_self_from_accounts、suggest_self_identities、
  apply_self_directions(sender∈self→out)、reunify(reset 只清自动连)。
- assist/proactive：自我会话被 proactive_sweep/openers 跳过；persons_overview 不含纯自我。
- web：/api/self、/api/reunify、人管理端点 shape（fake data）。

## Out of scope（下批）
persona 间**博弈/养心区**（养心护体第三支柱的交互）；跨 persona 分析；小红书/企微 adapter 接入
（登记结构已留 kind，adapter 另做）。
