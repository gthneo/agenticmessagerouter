# AMR v0.9.1 — PowerData wxid 映射 + 跨账号 person 合并 — 小 Design

**Date:** 2026-06-16 ｜ Status: approved ｜ 受 [[channel-vs-access-tool-router-trait]] /
[[self-identity-and-personas]] 约束。

## 问题
PowerData 会话只给联系人**名字**(备注/昵称)，无 wxid → 同一个人在 多账号/多工具 上无法按
稳定 id 合并（fullwechat 用 wxid 作 chat_id，powerdata 用 name）。目标：把同一个人(同一 wxid)
跨账号/工具合成**一个 person**（真机把 Shirley 在 代码班迪号 + 主号 [+ 仁兄号] 合一）。

## 设计
### 1. PowerData 取 wxid（owns: channels/powerdata.py）
- `get_contacts(account, query=name)` 返回 `wxid_xxx  备注: A  昵称: B`。新纯函数
  `parse_contact_wxid(text)` → 第一条的 wxid（无则 ""）。
- 适配器拉取时把会话 `chat_id` 解析成 **wxid**（`resolve_wxid(account,name)`：调 get_contacts
  → parse_contact_wxid；解析不到则回退用 name）。这样 powerdata 的 Shirley 会话 chat_id =
  其 wxid，和 fullwechat 对齐。

### 2. 跨账号 person 合并（owns: db.py）
`unify_by_wxid(conn)`：扫所有 **wechat** 会话，按 `chat_id` 分组（=wxid，排除 self_identities
里的 wxid 与明显非 wxid 的 name）。每组若有会话：取/建一个 person（已有该 wxid channel 的 person
优先；否则新建 id=`wx-<wxid 短哈希>`、name 取组内某会话名），把组内所有会话 `link_person` 到它，
并 `upsert_channel(wechat, wxid)`。返回 `{persons, linked}`。self 会话跳过。

### 3. 集成/真机（真名/真 wxid 仅在 .178 操作层，不入库）
.178 直连 LAN PowerData 重拉主号(account5)，会话 chat_id=wxid；确保 代码班迪号(fullwechat)
的同一联系人会话存在(wxid 对齐)；跑 `unify_by_wxid` → 两账号合一。再从另一实例重拉另一自有
账号(带 wxid 富化)push → 多账号合一。并复原此前被误解绑的那条会话。

## 测试（合成）
- powerdata: `parse_contact_wxid`（取首条 wxid / 空）。
- db: `unify_by_wxid` —— 两个不同账号、同一 wxid 的会话 → 合成一个 person、两会话都 linked；
  self wxid 的会话被跳过；不同 wxid 不误并。

## 安全/原则
仅读 get_contacts(只读)；person 自动建仅在有 wxid 强信号时（不靠名字盲建）；self 排除；
真名/wxid 不入库（测试合成）。
